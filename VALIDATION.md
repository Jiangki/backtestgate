# BacktestGate validation protocol

BacktestGate is still `VALIDATE`, not `BUILD-READY`. The current goal is to learn whether real Pine authors can complete an audit, act on a finding, and run it again. Fixtures, maintainer self-tests, stars, and downloads do not count as external workflow evidence (`W`).

## For testers

```bash
# Create a private local workspace.
python3 -m backtestgate init my-audit

# Follow my-audit/README.txt, then run the audit.
python3 -m backtestgate audit my-audit --output report.html

# Optional: create a receipt safe to attach to feedback.
python3 -m backtestgate audit my-audit \
  --share-output validation-receipt.json
```

The receipt contains:

- tool/schema version and a random `validation_id`
- PASS/WARN/BLOCK dimensions and rule IDs
- broad trade-count and coverage buckets
- whether exported net profit reconciled

It deliberately omits:

- Pine source and line evidence
- symbol, timeframe, paths, prices, P&L
- individual trades

The CLI never sends the receipt. Read it before sharing. Keep the generated `validation_id` unchanged if you fix findings and rerun; that allows repeat use to be recognized without an account.

## Feedback questions

1. Did you complete an audit using your own strategy export?
2. Which input or instruction caused friction?
3. Did any finding cause a code/configuration change?
4. Was any BLOCK incorrect? Include the minimum non-sensitive explanation.
5. Did you run the same `validation_id` again after a change?
6. Which missing check would block continued use?

Use the standalone repository's **Real audit feedback** issue form: https://github.com/Jiangki/backtestgate/issues/new/choose  
Never attach Pine source or raw CSV unless you independently intend to make them public.

## Maintainer evidence ledger

Record only user-authorized evidence. Do not infer a successful audit from a star, clone, issue view, or fixture run.

| Evidence ID | Date | External source/link | Receipt ID | Validation ID | Own export completed | Finding acted on | Repeat run | Permission/notes |
|---|---|---|---|---|---|---|---|---|
| — | — | No external evidence yet | — | — | — | — | — | Awaiting standalone release |

Counting rules:

- One externally submitted, internally consistent receipt plus explicit confirmation of using the user's own export counts as one completed audit.
- Multiple receipts with the same `validation_id` count as one user workflow and can establish repeat use.
- A report generated by maintainers, agents, tests, or bundled fixtures never counts as external `W`.
- A receipt is supporting evidence, not proof of identity or payment.
- Redact/delete accidental strategy or trade-data disclosures instead of copying them into the ledger.

## Current success and stop thresholds

- Continue validation: at least 5 external completed audits within 14 days of public release.
- Stronger signal: at least 2 validation IDs show a fix-and-rerun cycle.
- Stop or demote a rule: confirmed BLOCK false-positive rate exceeds 10%.
- Do not claim commercial demand until separate payment/budget evidence (`P`) exists.
